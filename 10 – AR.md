# День 10. AR для цифрового двойника

Сегодня я показываю еще один способ работать с цифровым двойником.

До этого мы смотрели на объект через OpenEgiz, Grafana и 3D-сцену. Теперь добавляем AR: открываем страницу в браузере, наводим камеру на marker и видим 3D лампу, которая берет свои текущие данные из цифрового двойника.

Главная мысль дня:

```text
цифровой двойник хранит состояние объекта
AR показывает это состояние через камеру
```

То есть AR здесь не отдельная игрушка. Это еще один интерфейс к тому же самому twin.

## Что мы собираем

Для демо у нас есть лампочка:

```text
summerschool:lightbulb-01
```

У нее есть features:

```text
brightness
power_consumption
voltage
temperature
```

Генератор обновляет эти значения через MQTT:

```text
lightbulb_generator.py
  -> Mosquitto
  -> OpenEgiz / Ditto
  -> summerschool:lightbulb-01
```

А AR-страница читает уже готовое состояние:

```text
AR browser
  -> https_server.py
  -> /api/lightbulb
  -> Ditto API
  -> brightness / power_consumption / voltage / temperature
```

Здесь важно объяснить студентам: браузер не ходит напрямую в Ditto.

Почему:

```text
браузеру для камеры нужен HTTPS
мы не хотим светить пароль ditto:ditto в frontend
могут быть проблемы с CORS
```

Поэтому между AR-страницей и Ditto стоит маленький proxy:

```text
https_server.py
```

Он делает две вещи:

```text
1. отдает AR-файлы
2. отдает API /api/lightbulb
```

А внутри сервера уже сам ходит в Ditto.

## Что должно быть готово

Перед началом у нас уже должен быть поднят OpenEgiz.

Проверяем:

```bash
kubectl get pods
```

И смотрим endpoints:

```bash
make endpoints
```

Для этого урока нам важны:

```text
Ditto:      30525
Mosquitto: 30511
Grafana:   30718
AR HTTPS:  30001
```

Порт `30001` должен быть открыт снаружи в firewall, иначе телефон или ноутбук просто не откроет AR-страницу.

## Проверяем twin лампочки

Сначала проверяю, что twin вообще есть и в нем лежат нужные features:

```bash
curl -s -u ditto:ditto \
  http://localhost:30525/api/2/things/summerschool:lightbulb-01/features \
  | python3 -m json.tool
```

Я ожидаю увидеть:

```text
brightness
power_consumption
voltage
temperature
```

Если эти поля есть и значения меняются, значит цифровой двойник живой и AR будет что показывать.

## Запускаем данные лампочки

Переходим в папку генератора:

```bash
cd ~/test/openegiz/data-generator
```

Если окружение еще не создавали:

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Запускаем генератор:

```bash
nohup python3 -u lightbulb_generator.py \
  --mqtt-host localhost \
  --mqtt-port 30511 \
  > lightbulb_generator.log 2>&1 &
```

Проверяем лог:

```bash
tail -f lightbulb_generator.log
```

После этого еще раз можно проверить Ditto:

```bash
curl -s -u ditto:ditto \
  http://localhost:30525/api/2/things/summerschool:lightbulb-01/features \
  | python3 -m json.tool
```

## Готовим AR-папку

AR-демо лежит здесь:

```bash
cd ~/test/openegiz/data-generator/AR
```

В папке есть:

```text
lampa.html
lampa.obj
lampa.patt
https_server.py
```

Объясняю это так:

```text
lampa.html       страница с AR
lampa.obj        3D модель лампы
lampa.patt       marker для распознавания камерой
https_server.py  HTTPS server и proxy к Ditto
```

## Создаем HTTPS сертификат

Камера в браузере нормально работает только через HTTPS. Поэтому на VPS создаем self-signed сертификат:

```bash
openssl req -x509 -newkey rsa:2048 -nodes \
  -keyout key.pem \
  -out cert.pem \
  -days 7 \
  -subj "/CN=<VPS_PUBLIC_IP>"
```

Например:

```bash
openssl req -x509 -newkey rsa:2048 -nodes \
  -keyout key.pem \
  -out cert.pem \
  -days 7 \
  -subj "/CN=34.118.106.123"
```

Важно: `cert.pem` и `key.pem` не пушим в GitHub. Их создаем локально на каждой VPS.

## Запускаем AR server

В этой же папке запускаем:

```bash
python3 https_server.py
```

Он слушает:

```text
0.0.0.0:30001
```

Проверяем API прямо на сервере:

```bash
curl -k https://localhost:30001/api/lightbulb | python3 -m json.tool
```

Если видим значения `brightness`, `power_consumption`, `voltage`, `temperature`, значит proxy работает и читает данные из Ditto.

## Открываем AR

На ноутбуке или телефоне открываем:

```text
https://<VPS_PUBLIC_IP>:30001/lampa.html
```

Например:

```text
https://34.118.106.123:30001/lampa.html
```

Браузер покажет предупреждение, потому что сертификат self-signed.

Нажимаем:

```text
Advanced -> Proceed
```

После этого разрешаем доступ к камере.

## Показываем marker

Внутри страницы используется marker:

```html
<a-marker type="pattern" url="lampa.patt">
```

То есть камера должна увидеть marker, который соответствует файлу `lampa.patt`.

Когда marker распознан:

```text
появляется 3D лампа
появляется панель с данными
данные обновляются через /api/lightbulb
```

И здесь я проговариваю главный смысл:

```text
мы смотрим не просто на 3D модель
мы смотрим на модель, связанную с цифровым двойником
```

В Grafana мы видим инженерный dashboard. В AR мы видим тот же объект через камеру. Источник состояния один и тот же: OpenEgiz.

## Быстрая диагностика

Если страница не открывается, проверяем, слушает ли server порт:

```bash
ss -ltnp | grep 30001
```

Если порт слушается на VPS, но страница снаружи не открывается, проверяем firewall:

```text
TCP 30001
source 0.0.0.0/0
```

Если API не работает:

```bash
curl -k https://localhost:30001/api/lightbulb
```

Если AR открылась, но лампа не появляется:

```text
камера должна быть разрешена
адрес должен быть именно https://
marker должен быть хорошо виден
свет и фокус камеры должны быть нормальные
```

Если браузер закешировал старую страницу:

```text
https://<VPS_PUBLIC_IP>:30001/lampa.html?v=2
```

Если данные на панели не обновляются, возвращаемся к Ditto:

```bash
curl -s -u ditto:ditto \
  http://localhost:30525/api/2/things/summerschool:lightbulb-01/features \
  | python3 -m json.tool
```

## Итог дня

Сегодня мы добавили AR как еще один слой поверх цифрового двойника.

Финальная цепочка:

```text
lightbulb_generator.py
  -> Mosquitto
  -> OpenEgiz / Ditto
  -> summerschool:lightbulb-01
  -> https_server.py
  -> AR browser
```

И это хорошая логика для объяснения:

```text
данные живут в OpenEgiz
визуализация может быть любой
Grafana, 3D, AR — это разные способы смотреть на один twin
```
