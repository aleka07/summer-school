# День 4. Создание первого цифрового двойника

Сегодня мы создаем наш первый цифровой двойник в OpenEgiz. Для первого примера берем простую лампочку.

Логика занятия такая: сначала создаем двойник в OpenTwins, потом подключаем к нему генератор данных. В реальном проекте вместо генератора было бы физическое устройство, контроллер или внешний сервис.

## Заходим в Grafana

Открываем Grafana на нашем сервере.

![[Pasted image 20260409104729.png]]

Дальше переходим в OpenTwins и открываем раздел `Twins`. Здесь мы будем создавать новый цифровой двойник.

![[Pasted image 20260409104948.png]]

Нажимаем `New Twin`.

## Создаем двойник лампочки

Заполняем основные поля:

```text
Namespace: summerschool
ID: lightbulb-01
Policy ID: basic_policy
Name: Лампочка
Description: Лампочка светит
Image URL: https://www.cahillheating.com/blog/which-light-bulb-is-best-for-every-room-of-your-house/
```

В `Attributes` добавляем простую информацию о том, где находится устройство:

| Key | Value |
|---|---|
| Комната | Лаборатория Цифровых Двойников |

![[Pasted image 20260409105505.png]]

Теперь добавляем `Features`. Это параметры, которые будут приходить в цифровой двойник:

```text
brightness
power_consumption
voltage
temperature
```

После этого нажимаем `Create`.

Вот здесь у нас появляется первый цифровой двойник: `summerschool:lightbulb-01`. Пока он просто описан в системе, но данных в нем еще нет.

## Зачем нужен генератор данных

В рамках летней школы мы не подключаем настоящую лампочку как IoT-устройство. Поэтому мы делаем генератор данных.

Важно понимать идею: генератор сейчас заменяет реальное устройство. Он будет вести себя как источник данных и отправлять значения в OpenEgiz через MQTT.

![[Pasted image 20260409110129.png]]

Генератор будет отправлять данные для двойника:

```text
summerschool:lightbulb-01
```

## Какие данные отправляет генератор

| Feature | Описание | Диапазон |
|---|---|---|
| `brightness` | Яркость | `0–800 lm` |
| `power_consumption` | Потребление | `0–60 W` |
| `voltage` | Напряжение | `210–235 V` |
| `temperature` | Температура | `25–85 °C` |

Когда лампочка включена, яркость и потребление растут. Когда выключена, яркость падает до нуля, мощность тоже падает, а температура постепенно снижается.

## Установка генератора

Переходим в папку генератора:

```bash
cd data-generator
```

Создаем виртуальное окружение:

```bash
python3 -m venv venv
source venv/bin/activate
```

Устанавливаем зависимости:

```bash
pip install -r requirements.txt
```

## Запуск генератора

Если мы находимся в корне проекта OpenEgiz, запускаем так:

```bash
source data-generator/venv/bin/activate
python3 data-generator/lightbulb_generator.py
```

По умолчанию генератор отправляет данные каждые несколько секунд. Если хотим изменить интервал, указываем `--interval`:

```bash
python3 data-generator/lightbulb_generator.py --interval 2
```

Остановить генератор можно через `Ctrl+C`.

## Запуск генератора в фоне

Если не хотим держать терминал занятым, запускаем в фоне:

```bash
python3 data-generator/lightbulb_generator.py --daemon
```

Логи будут здесь:

```text
data-generator/lightbulb_generator.log
```

Смотреть логи в реальном времени:

```bash
tail -f data-generator/lightbulb_generator.log
```

Остановить фоновый процесс:

```bash
python3 data-generator/lightbulb_generator.py --stop
```

## Управление состоянием on/off

Генератор поднимает HTTP API на порту `8090`. Через него мы можем включать и выключать лампочку.

Получить текущее состояние:

```bash
curl http://localhost:8090/state
```

Выключить:

```bash
curl -X POST -d '{"state":"off"}' http://localhost:8090/state
```

Включить:

```bash
curl -X POST -d '{"state":"on"}' http://localhost:8090/state
```

## Проверка данных в Ditto

Теперь проверяем, что данные реально попали в цифровой двойник:

```bash
curl -s -u ditto:ditto http://localhost:30525/api/2/things/summerschool:lightbulb-01/features | python3 -m json.tool
```

Если видим `brightness`, `power_consumption`, `voltage` и `temperature`, значит генератор работает, MQTT дошел до OpenEgiz, и цифровой двойник обновляется.

## Аргументы генератора

| Аргумент | По умолчанию | Описание |
|---|---|---|
| `--mqtt-host` | `localhost` | Адрес MQTT-брокера |
| `--mqtt-port` | `30511` | Порт MQTT-брокера |
| `--thing-id` | `summerschool:lightbulb-01` | ID цифрового двойника |
| `--interval` | `5` | Интервал отправки в секундах |
| `--http-port` | `8090` | Порт HTTP API |
| `--initial-state` | `on` | Начальное состояние |
| `--daemon` | — | Запуск в фоновом режиме |
| `--stop` | — | Остановка фонового процесса |
