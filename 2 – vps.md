# 2 день — создаем VPS в Google Cloud

Сегодня мы поднимаем сервер, на котором дальше будет работать OpenEgiz.

OpenEgiz — это не просто маленький сайт. Там будет несколько компонентов: Grafana, Ditto, Mosquitto, InfluxDB, MongoDB и другие сервисы. Поэтому нам нужен нормальный VPS, на котором это все сможет запуститься.

Сегодня наша цель:

```text
создать виртуальную машину
получить внешний IP
подключиться к серверу по SSH
подготовить доступ к портам для OpenEgiz
```

Архитектурную схему здесь пока не разбираем. Сегодня у нас инфраструктурный день: нам нужно получить сервер.

---

## 1. Заходим в Google Cloud

Открываем:

```text
https://console.cloud.google.com
```

Google Cloud обычно дает бесплатные кредиты на первый период. На скрине видно free trial:

```text
$300 credits на 90 дней
```

![[Снимок экрана 2026-04-08 в 16.38.24.png|677]]

Важный момент: если у вас сверху есть кнопка `Activate`, пока не нажимаем ее без необходимости. Нам для курса достаточно trial-режима.

И еще важнее: после завершения курса не забываем удалить виртуальную машину, чтобы случайно не получить списания.

---

## 2. Ищем Compute Engine

В верхнем поиске пишем:

```text
Compute Engine
```

![[Pasted image 20260408163914.png]]

Выбираем Compute Engine.

Это раздел Google Cloud, где создаются виртуальные машины.

---

## 3. Если Google просит включить двухфакторную аутентификацию

У некоторых аккаунтов может появиться сообщение, что доступ к Google Cloud заблокирован до включения 2-step verification.

![[Pasted image 20260408163940.png]]

Если такое появилось, нажимаем:

```text
Go to settings
```

и включаем двухфакторную аутентификацию.

После этого возвращаемся обратно в Google Cloud и снова открываем Compute Engine.

---

## 4. Переходим в VM instances

В левом меню открываем:

```text
Compute Engine → VM instances
```

![[Снимок экрана 2026-04-08 в 16.40.34.png]]

VM instances — это список наших виртуальных машин.

Если это новый аккаунт, список будет пустой.

---

## 5. Включаем Compute Engine API

Если Google Cloud просит включить Compute Engine API, нажимаем:

```text
Enable
```

![[Pasted image 20260408164137.png]]

Ждем, пока сервис включится.

Без этого Google Cloud не даст создать виртуальную машину.

---

## 6. Создаем виртуальную машину

Теперь нажимаем:

```text
Create instance
```

![[Pasted image 20260408164546.png]]

Откроется форма создания VM.

---

## 7. Настраиваем имя и регион

В начале задаем имя сервера.

Например:

```text
openegiz-vps
```

Регион выбираем:

```text
europe-central2 (Warsaw)
```

![[Pasted image 20260408165024.png]]

Название можно задать свое. Главное, чтобы потом было понятно, что это сервер для OpenEgiz.

---

## 8. Выбираем размер машины

Для наших целей нужен сервер минимум:

```text
4 vCPU
16 GB RAM
```

Выбираем серию:

```text
E2
```

Тип машины:

```text
e2-standard-4
```

![[Pasted image 20260408165039.png|697]]

Почему не меньше:

OpenEgiz поднимает несколько контейнеров. Если взять слабую машину, часть сервисов может не стартовать или будет очень медленно работать.

---

## 9. Настраиваем OS and storage

Переходим в раздел:

```text
OS and storage
```

Выбираем:

```text
Ubuntu 24.04 LTS
```

Важно выбрать именно x86/amd64 вариант.

Размер диска ставим:

```text
40 GB
```

![[Pasted image 20260408165343.png|697]]

40 GB нам достаточно для установки OpenEgiz, Docker, Kubernetes и базовых данных.

---

## 10. Data protection

В разделе Data protection выбираем:

```text
No backups
```

![[Pasted image 20260408165500.png|697]]

Для учебного стенда этого достаточно.

В реальном проекте backups обязательно нужны, но сейчас мы поднимаем временную учебную машину.

---

## 11. Networking

Переходим в раздел:

```text
Networking
```

Ставим галочки:

```text
Allow HTTP traffic
Allow HTTPS traffic
```

И включаем:

```text
IP forwarding
```

![[Pasted image 20260408165706.png]]

После этого нажимаем:

```text
Create
```

Ждем, пока виртуальная машина создастся.

---

## 12. Получаем внешний IP

После создания VM появится в списке.

![[Pasted image 20260408165913.png]]

Здесь нам важны две вещи:

```text
External IP
SSH
```

External IP — это адрес нашего сервера в интернете.

Например:

```text
34.118.106.123
```

У вас IP будет другой.

---

## 13. Самый простой способ подключиться

Самый простой вариант — нажать кнопку:

```text
SSH
```

справа от виртуальной машины.

![[Pasted image 20260409100714.png]]

Откроется браузерный терминал.

Если вы видите строку терминала, значит мы уже внутри сервера.

Для проверки можно выполнить:

```bash
ls
```

или:

```bash
htop
```

Если `htop` не установлен:

```bash
sudo apt update
sudo apt install htop
```

---

## 14. Нормальное подключение через SSH ключ

Браузерный SSH подходит для быстрого старта.

Но удобнее подключаться из своего терминала.

На своем компьютере создаем SSH ключ:

```bash
ssh-keygen -t rsa -f ~/.ssh/gcp_key -C "YOUR_USERNAME"
```

Где `YOUR_USERNAME` — имя пользователя на сервере.

Его можно увидеть в браузерном SSH. Например на скрине пользователь:

```text
developerbeck5
```

Когда команда спросит пароль для ключа, можно просто дважды нажать Enter.

Теперь выводим публичный ключ:

```bash
cat ~/.ssh/gcp_key.pub
```

Копируем весь текст, который начинается с:

```text
ssh-rsa
```

и заканчивается вашим username.

---

## 15. Добавляем SSH ключ в Google Cloud

В Google Cloud открываем:

```text
Compute Engine → Metadata → SSH Keys
```

Нажимаем:

```text
Edit → Add item
```

Вставляем публичный ключ и сохраняем.

После этого Google Cloud будет знать, что с нашего компьютера можно подключаться к VM.

---

## 16. Настраиваем быстрый вход

На своем компьютере открываем SSH config:

```bash
nano ~/.ssh/config
```

Добавляем:

```text
Host gcp
    HostName YOUR_EXTERNAL_IP
    User YOUR_USERNAME
    IdentityFile ~/.ssh/gcp_key
```

Пример:

```text
Host gcp
    HostName 34.118.106.123
    User developerbeck5
    IdentityFile ~/.ssh/gcp_key
```

Сохраняем:

```text
Ctrl+O → Enter → Ctrl+X
```

Теперь подключаемся:

```bash
ssh gcp
```

При первом подключении терминал спросит, доверяем ли мы этому серверу.

Пишем:

```text
yes
```

После этого мы попадаем на сервер.

---

## 17. Открываем порты для OpenEgiz

Позже OpenEgiz будет доступен через NodePort порты Kubernetes.

Обычно это диапазон:

```text
30000-32767
```

Чтобы Grafana и другие сервисы открывались из браузера, нужно добавить firewall rule в Google Cloud.

Открываем:

```text
Google Cloud Console → VPC network → Firewall
```

Создаем новое правило:

```text
Name: allow-nodeports
Direction: Ingress
Targets: All instances in the network
Source IP ranges: 0.0.0.0/0
Protocols and ports: TCP 30000-32767
```

Нажимаем:

```text
Create
```

После этого, когда мы установим OpenEgiz, Grafana будет доступна по внешнему IP и нужному порту.

Например:

```text
http://YOUR_EXTERNAL_IP:30718
```

---

## 18. Главная мысль дня

Сегодня мы создали сервер, на котором дальше будет работать OpenEgiz.

Что у нас должно быть в конце занятия:

```text
виртуальная машина в Google Cloud
Ubuntu 24.04
4 vCPU / 16 GB RAM
40 GB disk
external IP
SSH доступ
открытый диапазон портов 30000-32767
```

На следующем занятии мы уже зайдем на этот сервер и установим OpenEgiz.
